# GitHub Webhook Setup Guide for OpenShift

This guide explains how to set up GitHub webhooks to automatically trigger builds in OpenShift when you push code to your repository.

## Prerequisites

- OpenShift project deployed with BuildConfigs
- Admin access to GitHub repository: `https://github.com/bodarajeshkumar/idvalidation`
- OpenShift CLI (`oc`) installed and logged in

## Step 1: Get Webhook URLs from OpenShift

### For Backend BuildConfig

```bash
# Get backend webhook URL
oc describe bc backend | grep -A 1 "Webhook GitHub"
```

Output will look like:
```
Webhook GitHub:
  URL: https://api.your-cluster.com:6443/apis/build.openshift.io/v1/namespaces/data-retrieval/buildconfigs/backend/webhooks/<secret>/github
```

### For Frontend BuildConfig

```bash
# Get frontend webhook URL
oc describe bc frontend | grep -A 1 "Webhook GitHub"
```

Output will look like:
```
Webhook GitHub:
  URL: https://api.your-cluster.com:6443/apis/build.openshift.io/v1/namespaces/data-retrieval/buildconfigs/frontend/webhooks/<secret>/github
```

**Alternative method to get webhook URLs:**

```bash
# Backend webhook
oc get bc backend -o jsonpath='{.spec.triggers[?(@.type=="GitHub")].github.secretReference.name}'
oc get bc backend -o jsonpath='{.spec.triggers[?(@.type=="GitHub")].github.secret}'

# Frontend webhook
oc get bc frontend -o jsonpath='{.spec.triggers[?(@.type=="GitHub")].github.secretReference.name}'
oc get bc frontend -o jsonpath='{.spec.triggers[?(@.type=="GitHub")].github.secret}'
```

## Step 2: Add Webhooks to GitHub Repository

### 1. Go to GitHub Repository Settings

1. Navigate to: `https://github.com/bodarajeshkumar/idvalidation`
2. Click on **Settings** tab
3. Click on **Webhooks** in the left sidebar
4. Click **Add webhook** button

### 2. Configure Backend Webhook

**Payload URL:** Paste the backend webhook URL from Step 1

**Content type:** Select `application/json`

**Secret:** Leave empty (OpenShift handles this)

**SSL verification:** Enable SSL verification

**Which events would you like to trigger this webhook?**
- Select "Just the push event"

**Active:** Check this box

Click **Add webhook**

### 3. Configure Frontend Webhook

Repeat the same process for the frontend webhook URL:

1. Click **Add webhook** again
2. Paste the frontend webhook URL
3. Set Content type to `application/json`
4. Select "Just the push event"
5. Check Active
6. Click **Add webhook**

## Step 3: Test the Webhooks

### Test from GitHub

1. Go to **Settings → Webhooks**
2. Click on the webhook you just created
3. Scroll down to **Recent Deliveries**
4. Click **Redeliver** to test

### Test by Pushing Code

```bash
# Make a small change
echo "# Test webhook" >> README.md

# Commit and push
git add README.md
git commit -m "Test webhook trigger"
git push origin master
```

### Verify Build Started in OpenShift

```bash
# Check if builds started
oc get builds

# Watch build logs
oc logs -f bc/backend
oc logs -f bc/frontend
```

## Step 4: Verify Automatic Deployment

After the build completes, verify that pods are automatically updated:

```bash
# Check deployment status
oc get pods

# Check if new pods are running
oc rollout status deployment/backend
oc rollout status deployment/frontend
```

## Troubleshooting

### Webhook Not Triggering

1. **Check webhook delivery in GitHub:**
   - Go to Settings → Webhooks
   - Click on the webhook
   - Check "Recent Deliveries" for errors

2. **Verify BuildConfig triggers:**
   ```bash
   oc get bc backend -o yaml | grep -A 5 triggers
   oc get bc frontend -o yaml | grep -A 5 triggers
   ```

3. **Check OpenShift events:**
   ```bash
   oc get events --sort-by='.lastTimestamp'
   ```

### Build Fails

```bash
# Check build logs
oc logs -f build/backend-1
oc logs -f build/frontend-1

# Check BuildConfig
oc describe bc backend
oc describe bc frontend
```

### Pods Not Updating After Build

If pods don't automatically update after a successful build, you need to add image triggers to deployments:

```bash
# Manually trigger rollout
oc rollout restart deployment/backend
oc rollout restart deployment/frontend
```

Or add image triggers (already configured in deployment files):
```yaml
metadata:
  annotations:
    image.openshift.io/triggers: '[{"from":{"kind":"ImageStreamTag","name":"backend:latest"},"fieldPath":"spec.template.spec.containers[?(@.name==\"backend\")].image"}]'
```

## Manual Build Trigger

If you need to manually trigger a build without pushing to GitHub:

```bash
# Trigger backend build
oc start-build backend

# Trigger frontend build
oc start-build frontend

# Trigger with follow logs
oc start-build backend --follow
```

## Webhook Security

The webhook secret is automatically generated by OpenShift and is included in the webhook URL. GitHub uses this to verify requests are coming from your repository.

To regenerate the webhook secret:

```bash
# Delete and recreate BuildConfig
oc delete bc backend
oc apply -f openshift/backend-buildconfig.yaml
```

Then update the webhook URL in GitHub with the new URL.
